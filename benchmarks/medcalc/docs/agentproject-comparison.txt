=== AgentProject vs secretagent: Prompt Comparison ===
======================================================================

L0 BASELINE
----------------------------------------
AgentProject: Direct call_llm() with template:
  "You are a medical calculation assistant.
{formula_reference}
   Patient Note: {patient_note}
Question: {question}
   ...ANSWER: <number>"
  Formula reference dynamically extracted from calculators.py
  (Also had a bare baseline WITHOUT formulas for comparison)

secretagent: prompt_llm factory with baseline.txt template.
  Same structure. Formula reference dynamically from calculator_simple.py.
  DIFFERENCE: None substantive. Same framing, same info.

L1 SIMULATE / PTOOL
----------------------------------------
AgentProject: @ptool(model="deepseek-v3-0324", output_mode="freeform")
  Wraps function with docstring containing formula reference.
  ptool_framework generates its own prompt template.
  LLM sees function stub + docstring + input args.

secretagent: simulate factory with simulate.txt template.
  Same concept: function stub + docstring + "predict the output".
  DIFFERENCE: Different prompt template wording, but same structure.
  The ptool prompt and simulate prompt are both "predict function output"
  framings with the same formula information in the docstring.

L2 DISTILLED
----------------------------------------
AgentProject: @distilled(fallback_ptool=calculate_medical_value)
  Try Python extraction+calculation, fallback to L1 @ptool.

secretagent: direct factory with distilled_workflow().
  Try calculators.calculate(), fallback to simulate_medical_value.
  DIFFERENCE: The fallback in both cases is the L1 prompt.
  Python extraction logic is identical (same calculators.py code).

L4 PIPELINE
----------------------------------------
AgentProject: L4PipelineExperiment with 4 stages:
  1. identify_calculator_l4 — @ptool(structured) picks calculator
  2. extract_values_two_stage — raw call_llm() with reasoning+extraction
     * Included official calculator SOURCE CODE in the prompt
     * Had condition injection logic (clinical findings -> boolean flags)
  3. validate_extracted_values — Python validation + repair loop
  4. compute_with_python — official calculator implementations
  Fallback: L1 @ptool

secretagent: direct factory with pipeline_workflow():
  1. identify_calculator — simulate picks calculator
  2. _extract_values_two_stage — raw llm() with reasoning+extraction
     * Uses calculator docstring (not source code) as context
     * Simpler condition handling
  3. _validate_extracted_values — Python validation + single repair
  4. compute_official / compute_direct — Python computation
  Fallback: simulate_medical_value (= L1 prompt)

KEY DIFFERENCE in L4: AgentProject included the official calculator
Python source code in the extraction prompt, giving the LLM exact
parameter names and types. secretagent uses the calculator_simple
docstring instead, which has the same info but in natural language.
